História da Web[1]

(Voltar)

A World Wide Web, comumente conhecida como Web, foi concebida por Tim Berners-Lee em 1989. Sua criação visava permitir o compartilhamento de informações em documentos interligados através de hipertexto. Em 1991, Berners-Lee lançou o primeiro site e servidor web no CERN. O surgimento de navegadores como o Mosaic, em 1993, popularizou o acesso à Web. O boom da dot-com no final dos anos 90 viu um rápido crescimento de empresas baseadas na Web. A explosão das mídias sociais e serviços online no início dos anos 2000 mudou drasticamente a interação online. A Web 1.0 2.0 emergiu como uma fase de interatividade e colaboração, com sites como Wikipedia e YouTube. O advento de tecnologias como HTML5 e CSS3 permitiu uma experiência mais rica na Web. O surgimento de dispositivos móveis e aplicativos impulsionou o uso da Web móvel. A Web Semântica, uma visão de dados interconectados e significado compreensível por máquina, promete uma Web mais inteligente no futuro.

Comunicação Cliente x Server[2]

(Voltar)

    1. O que são clientes?

      Em um contexto cliente-servidor, os clientes são dispositivos ou softwares que solicitam serviços ou recursos a um servidor.. Eles podem ser navegadores da web, aplicativos móveis, programas de email, entre outros. Os clientes enviam solicitações ao servidor, que processa essas solicitações e envia de volta as respostas correspondentes. Geralmente, os clientes interagem com os servidores por meio de protocolos de comunicação como HTTP, FTP ou SMTP. Os clientes podem estar distribuídos em diferentes locais geográficos e se conectam aos servidores através da Internet ou de redes locais.

    2. O que são servers?

      Servidores são computadores ou sistemas de software que fornecem serviços, recursos ou dados para os clientes em um ambiente cliente-servidor. Eles respondem às solicitações dos clientes, processando-as e enviando as respostas correspondentes. Os servidores são projetados para serem confiáveis, escaláveis e capazes de lidar com múltiplas solicitações simultaneamente. Eles podem fornecer uma variedade de serviços, como hospedagem de sites, armazenamento de dados, gerenciamento de emails, entre outros.

    3. Linguagens de programação

      As linguagens de programação desempenham um papel crucial no contexto cliente-servidor, permitindo que os desenvolvedores criem aplicativos e serviços que interagem com os clientes e servidores. No lado do cliente, linguagens como JavaScript são usadas para criar interações dinâmicas e responsivas em navegadores web, enquanto em dispositivos móveis, linguagens como Swift para iOS e Kotlin para Android são comuns. No lado do servidor, linguagens como Python, Java, PHP e Ruby são usadas para criar a lógica de negócios e manipular dados, além de interagir com bancos de dados. Essas linguagens ajudam a garantir a comunicação eficiente e segura entre os clientes e servidores, possibilitando o desenvolvimento de aplicativos web e serviços robustos e escaláveis.

HTML na prática[3]

(Voltar)

  • Estrutura básica de um HTML

    1. O elemento <html> é o contêiner raiz de todo o documento HTML. Todos os outros elementos HTML devem estar contidos dentro deste elemento.
    2. O elemento <head> contém metadados, como o título da página, links para folhas de estilo CSS, scripts JavaScript, etc.
    3. O elemento <body> contém todo o conteúdo visível da página, como texto, imagens, links, formulários, etc. É dentro do que o conteúdo principal da página é colocado.

  • TAGS

    Tags como <h1> para títulos, <p> para parágrafos, <img> para imagens, <a> para links, entre outras, são usadas dentro do <body> para estruturar e apresentar o conteúdo.

  • Atributos

    • href: Usado dentro de <a>, o atributo href especifica o destino do link, que pode ser uma URL para outra página, um endereço de e-mail ou um identificador interno na mesma página.
    • src: Usado dentro de <img>, o atributo src especifica o caminho para a imagem que será exibida na página.
    • id e class: Esses atributos são usados para identificar elementos específicos dentro da página. O atributo id é único para cada elemento, enquanto o atributo class pode ser aplicado a vários elementos para aplicar estilos CSS ou comportamentos JavaScript.
